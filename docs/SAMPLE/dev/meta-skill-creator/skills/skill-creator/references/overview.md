## 1.1 Skillとは

Skillは、Claude（メインのオーケストレーター）が**「必要だ」と判断したときだけ**参照・起動する、自己完結型のパッケージ。

**核心的理解**:

> Skillは「メインに指示を追加する機構」ではなく、**コンテキストと作業領域をフェーズ単位で分離して汚さないための仕組み**

**位置づけ**:

| 観点                 | 説明                                         |
| -------------------- | -------------------------------------------- |
| 単なる保存プロンプト | ではない                                     |
| 実行設計の単位       | 段階的ロード、Task分離、リソース同梱を束ねる |

---

## 1.2 Skillが提供するもの

| 提供内容         | 説明                                                          |
| ---------------- | ------------------------------------------------------------- |
| 専門ワークフロー | 特定ドメイン向けの複数ステップ手順（実行設計）                |
| ツール統合       | 特定ファイル形式・コマンド・スクリプトを安全に扱う手順        |
| ドメイン専門知識 | 企業固有の用語、スキーマ、業務ルール（外部化前提）            |
| バンドルリソース | scripts（決定論処理）/ references（知識）/ assets（出力素材） |

---

## 1.3 責務境界

| 行うこと                                              | 行わないこと                                            |
| ----------------------------------------------------- | ------------------------------------------------------- |
| Skill内のワークフロー設計（段階・判断ポイント）を提示 | メイン会話の長期的な指揮統制（オーケストレーター責務）  |
| Task単位で必要な入力/出力/制約を明示                  | 他スキルの"自動実行"を強制（呼ぶ/呼ばないはメイン判断） |
| scriptsによる確実な実行（低自由度領域の外出し）       | README等の補助ドキュメント作成（不要）                  |
| テンプレで出力形式を標準化                            | すべての知識をSKILL.mdにベタ書き（referencesへ外部化）  |
