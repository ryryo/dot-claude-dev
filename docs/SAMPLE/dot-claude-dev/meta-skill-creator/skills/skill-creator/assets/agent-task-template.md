> 18-skills.md §3.3.2 準拠
> このテンプレートを使用して agents/ 内のTask仕様書を作成する
> **相対パス**: `assets/agent-task-template.md`

---

## クイックリファレンス

| セクション          | 責務                                         |
| ------------------- | -------------------------------------------- |
| 1. メタ情報         | 思考様式の参照ラベル（実在する専門家名）     |
| 2. プロフィール     | 背景・目的・責務（なぜその専門家か）         |
| 3. 知識ベース       | 参考文献と適用方法（詳細は references/ へ）  |

| 5. インターフェース | 入力/出力の契約（検証ルール・テンプレート）  |

## 使用方法

| ステップ | 内容                                               |
| -------- | -------------------------------------------------- |
| 1        | テンプレートをコピーし `agents/{{task}}.md` に配置 |
| 2        | `{{変数}}` を具体的な内容に置き換え                |
| 3        | 不要なセクションは削除可                           |
| 4        | references/ への参照は相対パスで記述               |

---

## 注意事項

### すべきこと

| 推奨事項                               | 理由                   |
| -------------------------------------- | ---------------------- |
| 実在する専門家名を参照ラベルとして使用 | 思考様式の明確化       |
| 適用方法を短く、詳細は references/ へ  | Progressive Disclosure |
| 入出力の検証ルールと欠損時処理を明記   | 堅牢なインターフェース |
| 出力テンプレートに `{{変数}}` を使用   | 一貫した出力形式       |
| Taskは単一責務に絞る                   | 分割統治で保守性向上   |

### 避けるべきこと

| 禁止事項                                   | 問題点               |
| ------------------------------------------ | -------------------- |
| 専門家本人を名乗る／発言を捏造             | 倫理的問題           |
| 長い知識本文をTask仕様内に埋め込む         | 肥大化、保守困難     |
| references/ へのリンクなしで詳細知識を記述 | 重複、不整合のリスク |
| 曖昧な条件記述（「必要に応じて」等）       | 再現性の低下         |
| 複数責務を1つのTaskに同居させる            | 仕様の曖昧化         |
