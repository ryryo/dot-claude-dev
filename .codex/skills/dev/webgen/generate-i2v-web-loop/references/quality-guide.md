## 基本設計

Web用i2vでは、映像的な派手さより次を優先する。

1. 構図と情報階層を維持する。
2. カメラを固定する。
3. 動かす対象を1〜3種類に絞る。
4. 4秒程度で一往復させる。
5. 最終成果物を無音にする。
6. `prefers-reduced-motion`向けに元画像を残す。

## 入力画像

- 寸法とアスペクト比を記録する。
- 文字、ロゴ、UIを可能なら動画生成対象から分離する。
- 細線や小さな文字が多い場合、最低解像度は構図確認用と割り切る。
- 透過画像はモデル側で背景が補完される可能性があるため、掲載時の背景色で事前に合成する。

## ループ設計

同じ画像を開始・終了に指定しても完全なループは保証されない。次の動きを選ぶ。

- 光のパルス
- 表示灯の明滅
- 呼吸、まばたき
- 布、髪、葉の小さな往復
- 機械部品の短い一往復

一方向へ流れ続ける物体、大きな移動、歩行、場面転換は境界が目立ちやすい。

4秒を既定とする理由:

- 3秒より動きを一往復させやすい。
- 5秒以上よりレイアウトドリフトを抑えやすい。
- 最低料金で比較しやすいモデルが多い。

## 文字とロゴ

文字をモデルに再生成させない。次を明記しても中間フレームで変形する可能性がある。

- typographyを完全維持する。
- text animationを禁止する。
- new text、caption、logo changeを禁止する。

厳密な維持が必要なら、動画から文字領域を除き、HTML/CSSの静止レイヤーとして後から重ねる。

## 実測から得た注意点

2026-07-20に同一の4:3画像、同一プロンプト、同一開始・終了画像、4秒で7モデルを比較した。

- 全モデルが約4.04秒、24fps、無音で出力された。
- 指定解像度と実ピクセル寸法は一致しない場合があった。
- 4:3指定でも数ピクセル程度の比率差が発生したモデルがあった。
- 先頭・末尾SSIMはKling O3 StandardとKling V3 Standardが約0.99だった。
- Seedance Standardは見た目が近くても境界SSIMが低い出力があり、モデル名だけでループ品質を判断できなかった。
- Proモデルはファイルサイズが大きくても、必ずしも境界品質が高いとは限らなかった。

したがって、価格やモデル階層ではなく生成物単位で検査する。

## 採用基準

- 文字が全フレームで読める。
- 主要オブジェクトの数、位置、形が変わらない。
- カメラ移動、クロップ、パース変化がない。
- 5回以上の連続再生で境界が気にならない。
- 入力と出力の比率差が1%以内。
- 音声ストリームがない。
- 実表示サイズに対して解像度と容量が過剰でない。

先頭・末尾フレームが近くても、境界で速度が急変する動画は不採用にする。

採用後のループ補修は [loop-repair.md](loop-repair.md)、実表示サイズに応じた形式・解像度・HTML実装は [web-delivery.md](web-delivery.md) を参照する。
