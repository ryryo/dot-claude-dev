# 品質ガイド

## 目次

- 基本設計

- 代表フレーム確認

## 基本設計

人物の短尺メッセージ動画では、映像的な派手さより次を優先する。

1. 人物IDが保たれている。
2. カメラと構図が固定されている。
3. 動きが1メッセージに対応している。
4. 顔、手、持ち物が破綻しない。
5. 最終成果物が無音である。
6. テロップは後処理またはHTMLで管理できる。

## 人物動画の採用基準

- 顔が別人化していない。
- 表情変化が自然で、過剰に笑いすぎない。
- 指が増えない、手が溶けない、財布や資料が変形しない。
- 背景が流れない。
- カメラのズーム、パン、クロップがない。
- テロップ、ロゴ、UI、字幕が新規出現しない。
- 音声ストリームがない。
- Web掲載サイズに対して解像度と容量が過剰でない。

## 代表フレーム確認

`inspect_videos.py`が`_inspection/<model>-contact-sheet.jpg`を作る。ここで次を見る。

- 先頭、途中、末尾で顔と手が安定しているか。
- 持ち物が同じ形で残っているか。
- 目や口の動きが大きすぎないか。
- 背景や服の境界が波打っていないか。

SSIMは補助指標であり、人物動画では先頭末尾が一致しなくても問題ない場合がある。固定カメラ短尺動画として自然なら採用候補にできる。

## よくある失敗

- スタジオ素材のような嘘っぽい質感になる。
- 動画生成で笑顔が強くなりすぎる。
- 指や財布の縁が揺れる。
- カメラがわずかにズームする。
- Seedance系で人物入力が拒否される。

入力静止画の段階で、ポーズを単純化し、手と持ち物を大きく見せすぎないと安定しやすい。
