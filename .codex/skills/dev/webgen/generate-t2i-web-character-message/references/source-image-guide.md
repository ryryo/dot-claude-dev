## 入力素材の優先順位

1. 採用済みの動画用1枚絵
2. キャラクターシート + 構図参考画像
3. 人物参考画像 + 構図要件
4. 要件テキストだけ

i2vへ直接渡すのは、必ず「動画化したい1枚絵」にする。キャラクターシートそのものをi2vへ渡さない。

## キャラクターシートがある場合

キャラクターシートは人物ID、髪型、服装、年齢感、表情差の参照として使う。動画化する前に、次を満たす1枚絵を作る。

- 1人だけが写っている。
- 動画内で使うポーズ、持ち物、表情が既に入っている。
- テロップ、ロゴ、UI、不要な文字がない。
- 画角は掲載比率に近い。
- キャラクターシート的な複数ポーズや注釈が残っていない。

## 参考画像だけがある場合

参考画像は構図、人物位置、表情、スマホ画質、照明、ポーズの参照として使う。元画像の人物IDを使うか、新しいキャラクターに置き換えるかを明示する。

素材サイト風の均一なスタジオ写真より、動画フレームに近い次の質感を優先する。

- iPhone動画の一コマのような少し粗い解像感
- 室内のフラットで少し不均一な照明
- 適度な圧縮感
- 背景と人物の境界が完璧すぎない
- 肌や服の過度なレタッチがない

## 入力画像がない場合

まず画像生成で人物の静止画を作る。必要ならキャラクターシートを先に作る。

推奨順:

1. 主要人物の設定を決める。
2. 通常デフォルメ、ちび、表情差などが必要ならキャラクターシートを作る。
3. 実写動画用なら、動画化する1枚絵を別途作る。
4. i2vへ渡す前に、文字とロゴを除外する。

## i2v前の静止画基準

- 人物の顔、手、持ち物がフレーム内で明確。
- 人物の視線、手、持ち物、画面や紙面の向きが、1枚の中で同じ行為として成立している。
- 情報が出ている面が、実際に画面、紙面、ラベルなどの表示面である。端末や商品の背面、側面、反射、影に情報が出ている画像は採用しない。
- カメラ位置から見えないはずの画面内容、資料内容、相手の顔を無理に見せていない。
- 動かしたくない要素が多すぎない。
- 既にメッセージが1つに絞れている。
- 下部テロップを入れる場合、顔や持ち物を隠さない余地がある。
- 実写風なら、過度なAI光沢、スタジオ素材感、完璧すぎる背景を避ける。

## i2vモデル選択上の注意

Seedance系は、実写人物風の生成画像に対して、肖像や私的情報の検証で結果取得時に拒否される場合がある。原因をメタデータだけと決めつけず、人物写真風入力そのものが拒否対象になる可能性を考慮する。

Seedance系で拒否された場合、同じ条件で再送を繰り返さず、Kling O3 StandardまたはKling V3 Standardへ切り替える。
