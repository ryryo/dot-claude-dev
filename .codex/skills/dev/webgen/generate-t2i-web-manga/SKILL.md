# Web漫画画像生成

## ゴール

LP・Webサイトへ掲載できる短い漫画画像を生成する。漫画候補の分析、台本、プロンプト提示だけで完了しない。

ユーザーが「作って」「生成して」と依頼した場合は、`$imagegen`を使って画像を生成し、結果を目視確認し、必要なら修正生成する。「プロンプトだけ」と明示された場合に限り、生成を行わずプロンプトを返す。

このスキルで扱う漫画の基本単位は、`1画像1ストーリー`とする。1枚の中に複数の論点や別々のストーリーを詰めず、短い時間変化、感情変化、理解の変化を1本だけ描く。

画像生成前に、必ず[Web漫画画像用プロンプト](references/adapted-manga-prompt.md)を最後まで読む。出力形式、画風、コマ割り、文脈、必須条件、キャスト役割、原稿情報を揃え、案件固有の原稿、掲載箇所、キャスト設定を埋めて使う。

## 対象

- 読者の悩みや行き詰まりへの共感
- 一般論、AI原稿、既存手段への違和感
- 相談、取材、確認、承認などの短い対話
- Before/Afterや行動変化
- サービスの価値を、人物の反応と場面で理解させる漫画
- LP内に差し込む1コマ、2コマ、3コマ、4コマ漫画

## 入力

ユーザーが任意指定できる定型項目は次の3つまでとする。

```yaml
text_mode: include
aspect_ratio: 4:3
panel_count: 3
```

- `text_mode: include`: セリフ、短い地の文、短い見出しを入れる。漫画の既定値とする。
- `text_mode: none`: 画像内に文字を入れない。HTML側でコピーを載せる、または文字崩れを避けたい場合に使う。
- `aspect_ratio`: 明示指定、実際の掲載枠、`4:3`の順で決める。掲載枠の根拠がなければ`4:3`とする。
- `panel_count`: 明示指定がなければ`3`とする。LP差し込みでは1、2、3、4コマまでを標準とし、5コマ以上はユーザーが明示した場合だけ使う。

配色は固定しない。対象ページ、参照画像、既存キャラクター、掲載文脈になじむ色を使う。ユーザーや画像計画が配色を指定している場合は、その指示を優先する。

## 出力規約

Image Gen共通の出力契約は [imagegen-core](../imagegen-core/SKILL.md) を読む。保存先、候補画像、修正生成、メタデータ、報告形式はimagegen-coreの規約に従い、このスキル側で再定義しない。

## 生成手順

### 1. 必要な資料だけを読む

対象の原稿、掲載セクション、画像計画、キャスト表、人物リファレンス、既存デザインを確認する。すでに資料から分かることをユーザーへ聞き直さない。

人物が登場する場合は、既存キャストや人物リファレンスを優先する。画像ごとに新しい人物を増やさない。ここにない人物が必要な場合は、生成前に役割、登場理由、外見の固定点を明確にする。

### 2. ストーリーを決める

原稿から、この漫画で見せる1つの短いストーリーラインを決める。読者に伝える変化、気づき、自分ごと化しやすい反応ポイントのうち1つを主軸にし、コマへ分解する。

必要な場合は、まず原稿や画像計画から漫画AIへ渡す背景情報を作る。この段階では完成台本を書き切らず、読者の悩み、状況、人物関係、変化点、避ける誤解を網羅的に整理する。

### 3. コマ割りと表現を決める

各コマに、場面、登場人物、表情、視線、手元、小道具、セリフまたは無言の動作を割り当てる。説明だけでなく、表情や姿勢の変化で読ませる。

LP掲載用では、過度なギャグ、煽り、絶望、成功誇張を避ける。読者が自分ごと化できる程度の現実味を保つ。

### 4. 生成プロンプトを組み立てる

[Web漫画画像用プロンプト](references/adapted-manga-prompt.md)を土台にし、ストーリー、コマ数、コマごとの場面、人物同一性、画風、文字の扱い、アスペクト比、避ける表現を具体的に含める。

「読みやすい漫画」「高品質」などの抽象語だけで済ませない。誰が、どのコマで、何を見て、どんな表情になり、何に気づくかを書く。

### 5. 画像を生成する

`$imagegen`の手順に従って生成する。参照画像がある場合は事前に確認し、必要な画像だけを渡す。

複数漫画を依頼された場合は、指定順またはLPの掲載順で全件生成する。最初の1枚だけを試作して止めない。ただし生成失敗が連続する場合は、原因を修正してから続ける。

### 6. 目視確認して修正する

生成画像を必ず開き、次を確認する。

- 1枚の漫画として1つのストーリーが読めるか
- コマ順、視線誘導、時間経過が自然か
- 登場人物の顔、年代、髪型、体格、服装が参照と一致しているか
- 表情や姿勢が、悩み、違和感、気づき、行動変化を伝えているか
- セリフや短い文字が崩れていないか
- 細かいUI文字、読めない看板、透かし状要素が混入していないか
- 原稿、掲載箇所、注意事項と矛盾していないか
- 実在しない成果、口コミ、受賞、保証を表現していないか
- PCとモバイルの想定トリミングで重要なコマや人物が欠けないか

問題があれば、問題箇所を明示して修正生成する。生成できたという理由だけで採用しない。

### 7. 保存して報告する

採用画像、候補画像、メタデータをimagegen-coreの出力契約に従って整理する。

完了時はimagegen-coreの報告形式に従い、漫画固有の主要オプションとして`text_mode`、`aspect_ratio`、`panel_count`を含める。
