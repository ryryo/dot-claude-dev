# 漫画画像生成ルール

このファイルは、LP・Webサイト用の短い漫画画像を生成するときの詳細ルールとする。画像生成時は全文を読み、該当案件の原稿、掲載箇所、キャスト設定、参照画像を埋めて使う。

## 入力変数

```text
【入力変数: テーマ】
依頼文、原稿、掲載セクションから導いた、この漫画で描く1つの短いストーリー。

【入力変数: text_mode】
include または none。未指定は include。

【入力変数: aspect_ratio】
明示指定、実際の掲載枠、4:3の順で決める。未指定かつ掲載枠不明は4:3。

【入力変数: panel_count】
未指定は3。LP差し込みでは1、2、3、4コマを標準にする。
```

ここから下は画像生成AIと制作エージェントへの指示であり、画像内には表示しない。この指示文、見出し、例文、ルール説明を画像内テキストとして使わない。

## 基本思想

- 漫画は、説明図ではなく短い時間変化を見せる画像として作る。
- 1画像には1つの短いストーリーラインだけを入れる。
- 1コマごとの情報量を減らし、表情、視線、手元、小道具で状況を読ませる。
- LP内では、本文の説明を繰り返すのではなく、読者の共感、違和感、気づき、行動変化を補助する。
- 読者が「これは自分の状況に近い」と感じる現実味を優先する。

## 漫画の型

必要に応じて、次の型から1つだけ選ぶ。複数の型を1枚に混ぜない。

### 悩み連鎖

用途は、読者共感、問題提起、行き詰まりの可視化。

- 1コマ目: 通常業務や日常の負荷を見せる。
- 2コマ目: 誰かからの質問、未完了タスク、判断の詰まりを見せる。
- 3コマ目: 本質的な悩みや違和感に気づく。

### 違和感発見

用途は、一般論、AI原稿、既存手段との差を見せる。

- 1コマ目: 一見うまくできた成果物を見る。
- 2コマ目: 自社らしさ、一次情報、判断の欠落に気づく。
- 3コマ目: 別の進め方が必要だと理解する。

### 役割分担

用途は、サービス理解、取材や編集の流れ、誰が何を担うかの理解。

- 1コマ目: 顧客が話す、質問に答える、素材を出す。
- 2コマ目: 編集者や担当者が整理する。
- 3コマ目: 記事、方向性、次の行動にまとまる。

### Before/After

用途は、状態変化、導入前後、心理的負担の軽減。

- 1コマ目: 抱え込んでいる状態。
- 2コマ目: 任せる、話す、整理される転換点。
- 3コマ目: 本来の業務や判断に集中できる状態。

## コマ数

- 1コマ: 強い一場面やリアクションだけを見せる。
- 2コマ: Before/Afterや対比を見せる。
- 3コマ: 悩み、気づき、変化を見せる標準構成。
- 4コマ: 起承転結が必要な場合だけ使う。
- 5コマ以上: LP差し込みでは原則使わない。ユーザーが明示した場合だけ使う。

## 人物とキャスト

- 既存キャスト、人物リファレンス、キャラクターシートがある場合は、それを正とする。
- 同じ人物は、全コマで顔、年代、髪型、体格、服装を維持する。
- 写真カテゴリの人物を漫画化する場合も、同じ人物だと分かる外見特徴を残す。
- 登場人物を増やすほど読みにくくなるため、標準は1〜2人にする。
- 3人以上は、会議、チーム、対立構造など、ストーリー上の必然がある場合だけ使う。
- AI、システム、記事、検索、サービスは人物化せず、画面、書類、アイコン、小道具として表す。

## 画風

- LP・Webサイトになじむ、清潔で読みやすいビジネス漫画にする。
- 過度なギャグ漫画、激しい集中線、巨大な汗、極端な怒り顔、泣き崩れ、成功ポーズは避ける。
- 線は明瞭にし、人物の表情と手元が分かるようにする。
- 背景は必要最小限にし、オフィス、机、PC、ノート、付箋、資料など文脈を示す小道具を使う。
- 写真風、アイソメ図解、フルCG、アニメポスター風に寄せない。ユーザーが明示した場合だけ変更する。

## 配色

- 固定パレットは持たない。
- 対象ページ、参照画像、既存キャラクター、掲載文脈になじむ色を使う。
- 色数を増やしすぎず、人物、背景、注目箇所の関係が読みやすい配色にする。
- ユーザーや画像計画が配色を指定している場合は、その指示を優先する。

## 文字

### text_mode: include

- セリフ、短い地の文、短い見出しを入れてよい。
- 1吹き出しは短くし、長文説明を入れない。
- 画像生成で文字が崩れやすい場合は、セリフ量を減らすか、文字なしで再生成する。
- Markdown記号、内部指示、読めない疑似UI文字を入れない。
- 価格、契約条件、統計、出典、保証、注意書きなど正確性が重要な情報は原則HTMLに残す。

### text_mode: none

- セリフ、看板、画面内文章、端の小文字を含め、画像内に文字を描かない。
- 表情、視線、手元、小道具、コマ順だけでストーリーが読めるようにする。

## 生成プロンプトに必ず含めるもの

- 掲載箇所と漫画の役割。
- 1つの短いストーリーライン。
- `panel_count`とコマの読む順番。
- 各コマの場面、人物、表情、視線、動作、小道具。
- 登場人物の固定外見と、参照画像から維持する特徴。
- 画風、線、背景、色の方向性。
- `text_mode`と、文字を入れる場合のセリフ量。
- `aspect_ratio`。
- 避ける表現。

## 禁止・注意

- 1枚に複数ストーリーを入れない。
- 画像ごとに別人化させない。
- 社員や顧客を無能、怠慢、悪者として描かない。
- AIを危険な存在や敵として人格化しない。
- 実在しない成果、口コミ、受賞、保証を漫画内で作らない。
- 実在顧客の記録写真や体験談に見える演出にしない。架空の利用イメージとして扱う。
- 細かい画面UIや資料本文を読ませる構図にしない。
- LP本文、契約条件、注意事項と矛盾する展開にしない。

## 最終確認

- 1画像1ストーリーになっているか。
- コマ順と時間経過が自然か。
- 人物の同一性が保たれているか。
- 表情、視線、手元、小道具で悩みや気づきが伝わるか。
- 文字が崩れていないか。崩れている場合は修正生成するか、`text_mode: none`へ切り替える。
- 掲載セクションの本文と重複しすぎず、補助として機能するか。
- モバイル幅でコマや人物が小さくなりすぎないか。
