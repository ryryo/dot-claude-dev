# キャラクターシート画像生成

## ゴール

参照写真を完成見本ではなく人物資料として扱い、LP、広告、記事、漫画、場面写真の後続生成で同じ人物を再現できる基準画像を作る。

単に元写真へ似せるだけで完了しない。本人性を保ちながら、依頼で指定された髪型、身だしなみ、服装、表情、光を整え、複数アングルで破綻しない制作素材へ仕上げる。

ユーザーが「作って」「生成して」「直して」と依頼した場合は、`$imagegen`を使って実際に画像を生成し、保存と目視確認まで行う。プロンプトだけを求められた場合だけ生成を省略する。

## 基本原則

### 参照画像を役割別に使う

複数の参照画像を平均化しない。生成前に各画像の役割を決める。

- **本人性の基準**: 顔の骨格、目、鼻、口、輪郭、年齢感、眼鏡などを採用する画像
- **部分意匠の基準**: 髪型、眉、服装、体格、表情など特定要素だけを採用する画像
- **撮影品質の参考**: 光、色、背景、レンズ感などだけを参考にする画像
- **除外対象**: 一時的な寝癖、乱れた襟、青髭、肌のテカリ、歪みなど引き継がない要素

プロンプトには「どの画像から何を採用し、何を採用しないか」を明記する。

### 本人性と整身を分離する

本人性を守る要素と、制作上整える要素を混同しない。

- 守る: 顔の骨格、目鼻口の比率、輪郭、年齢感、本人固有の印象
- 整える: 髪の収まり、眉、髭跡、肌の反射、襟、前立て、袖、服の皺、姿勢
- 指示がなければ変えない: 体格、服の種類、眼鏡、髪色、年齢、表情の方向性

整身を理由に別人化、美形化、若返り、過剰な小顔化を行わない。

### キャラクターシートとして設計する

1枚の良いポートレートではなく、後続画像で参照しやすいシートを作る。指定がなければ、同一背景・同一光・同一衣装で次を含める。

- 正面の頭部から胸元
- 左右いずれかの3/4ビュー
- 反対側の3/4ビュー
- 横顔
- 腰上または全身

各パネルで顔、髪、眼鏡、眉、服の仕様を揃える。余白を確保し、人物同士を重ねない。文字、ラベル、装飾、透かし状要素は、明示依頼がない限り入れない。

### 実写とイラストを混ぜない

実写版とイラスト版の両方が必要な場合は、同時に曖昧な中間表現を作らない。次の順序で分けて生成する。

1. 参照写真から、身だしなみを整えた実写シートを本人性の正本として完成させる
2. 承認できた実写シートを参照し、顔、髪、眼鏡、服、年齢感を固定したイラストシートを別に作る
3. イラストでは画風、線、陰影、頭身を全ビューで固定する

ユーザーがイラストだけを求める場合も、写真資料から人物仕様を先に言語化してから画風を適用する。画風を顔立ちより優先して別人化させない。

## 制作手順

### 1. 資料を確認する

参照画像を`view_image`で開き、人物と各要素を目視する。ファイル名やユーザーの説明だけで判断しない。

対象ページ、画像計画、既存キャスト設定があれば必要な範囲だけ読み、次を内部で固定する。

```yaml
identity_source: 顔と本人性の基準画像
detail_sources:
  hair: 髪型の基準画像
  wardrobe: 衣装の基準画像または記述
fixed_traits: 絶対に維持する特徴
grooming_edits: 制作向けに整える特徴
photo_style: 光、背景、レンズ感、色温度
sheet_views: 必要なアングル
do_not_change: 変更禁止項目
```

資料から分かる内容をユーザーへ聞き直さない。本人性の基準が曖昧で別人化の危険が高い場合だけ確認する。

### 2. 完成時の人物仕様を先に決める

生成前に完成仕様を一度統合する。元画像に対する小修正の羅列だけにしない。

- 顔: どの画像を本人性の正とするか
- 髪: 長さ、分け目、ウェーブ、毛量、前髪、耳周り、襟足
- 眉: 太さ、濃さ、形、毛流れ、手入れの程度
- 髭: 有無、剃毛後の見え方、青みを残すか
- 肌: 毛穴、艶、色、レタッチ強度
- 服: 種類、色、襟、首元、前立て、袖、フィット
- 写真: 背景、光源、色温度、コントラスト、被写界深度
- 表情: 中立、微笑みなど各ビューの差

「素材として整える」という目的から、参照写真にある偶発的な乱れを完成仕様へ持ち込まない。

### 3. 生成プロンプトを組み立てる

プロンプトを次の順序で具体化する。

1. 用途: 制作向け実写キャラクターシート
2. 本人性: 顔の基準と維持項目
3. 部分統合: 別資料から採用する髪型など
4. 身だしなみ: 眉、髭、肌、服の整え方
5. 撮影: 背景、照明、カメラらしい階調
6. レイアウト: アングル、余白、パネル数
7. 禁止事項: 別人化、過剰補正、文字、破綻

「自然に」「高品質に」「整えて」だけで済ませない。何を残し、どこをどう整え、何を避けるかを書く。

### 4. 自然なスタジオ写真として指定する

実写では、カメラ名だけを書いて自然さを期待しない。ユーザーが特定機材を指定した場合は「その機材で撮影したような表現」として扱い、次も併記する。

- 大きなソフトボックスによる柔らかいキーライト
- 弱いレフ板のフィル
- 中立的な昼光色
- なだらかなハイライトの減衰
- 正確で落ち着いた肌色
- 毛穴を残した自然な肌理
- 過度に鋭くない解像感
- 控えめなコントラストと被写界深度

額、鼻、頬、唇の油膜状の反射、プラスチック肌、HDR風の局所コントラスト、過剰なオレンジ肌、美肌フィルター、生成画像特有の均一なテカリを明示的に除外する。

### 5. 身だしなみを具体的に整える

#### 髪

別画像の髪型を採用する場合は、長短だけでなく、分け目、トップの高さ、ウェーブの強さ、前髪の方向、耳周り、襟足を指定する。全ビューで同じカットとして成立させる。

#### 服

服装を維持する場合も乱れまで維持しない。次を確認する。

- 襟が左右とも自然に寝ている
- 襟先と首元が不自然に捻れていない
- Tシャツの襟ぐりが平らで中央にある
- 前立てが曲がらず、深い皺や引きつれがない
- ロールアップした袖が左右で整っている
- 清潔なビジネスカジュアルに見える

フォーマル化の指示がなければ、スーツ風、硬い仕立て、過度なプレスへ変えない。

#### 髭と肌

剃毛済みの指定では、上唇、顎、頬、顎線の青灰色の髭跡と点状の無精髭を除く。ただし毛穴や成人男性の自然な肌理は残す。髭を消す処理で蝋人形のような肌にしない。

#### 眉

「眉毛サロンで整えた」は、原則として描き足すことではない。次の処理として解釈する。

- 長い毛を切る
- 密集箇所を間引く
- 毛流れを梳かす
- 眉間と輪郭外の余分な毛を除く
- 左右の高さ、長さ、形を自然に揃える
- 男性らしい直線寄りの形に、ごく弱い自然な弧と細い眉尻を作る

元の毛量を基準とし、濃く塗る、疎な箇所を面で埋める、太くする、真っ黒にする、輪郭をスタンプ状にする、眉メイクやアートメイクのようにすることを避ける。

「濃さを均等にする」は、濃い箇所を間引いて視覚密度を揃える意味で扱う。ユーザーが明示しない限り、薄い箇所へ大量に描き足して均等化しない。

### 6. 生成して目視確認する

`$imagegen`の手順に従って参照画像を渡し、生成する。生成後は必ず成果物を`view_image`で開き、通常表示と顔が判断できる大きさで確認する。

次をすべて確認する。

- 全パネルが同一人物に見える
- 顔の骨格と年齢感が本人性の基準から逸脱していない
- 指定した髪型が全方向で同じカットとして成立する
- 眉、髭、肌、襟、袖の修正が反映されている
- 修正が通常表示で認識できるが、過剰ではない
- 肌に青髭、人工的な光沢、プラスチック感がない
- 服が清潔に整っているが、別の服へ変わっていない
- 眼鏡、耳、手、服のボタン、パネル境界に破綻がない
- 意図しない文字や記号がない

「生成できた」ことを合格条件にしない。依頼された差が見えなければ未反映として修正する。

### 7. 修正を安全に反復する

連続した生成画像だけを次の入力に使い続けると、顔、肌、服、光が少しずつ変わる。修正ごとに次を行う。

1. 最後に承認できた安定版を基準として残す
2. 本人性が揺れそうなら元の本人性資料も再度参照する
3. 直す項目と、絶対に変えない項目を両方書く
4. 修正が弱すぎた場合は、抽象語を増やさず可視的な差を具体化する
5. 修正が強すぎた場合は、強い版を薄めず、安定版からやり直す

眉の例では、薄い修正から太く濃い眉へ振り切れた場合、濃い版を再編集せず、自然な安定版へ戻って「描き足さず、間引きとカットで整える」と指定する。

## 失敗条件

次のいずれかがあれば採用しない。

- 元写真をほぼそのまま並べただけで、指定された整身が反映されていない
- 複数資料を平均化して本人性が薄れた
- ビューごとに髪型、眉、眼鏡、年齢、輪郭が変わる
- 襟、首元、袖、前立てが乱れている
- 青髭を消した代わりに肌が無毛・無孔のプラスチック状になる
- 光源の位置や肌色がパネルごとに違う
- 眉の修正が見えない
- 眉を均一化するために太く濃く描き足している
- カメラ名だけ指定し、AI特有のテカリやHDR感が残っている
- 修正のたびに顔が別人へ近づいている

## 保存と版管理

プロジェクト内の保存規則へ従う。規則がなければ、採用候補と過程が区別できるディレクトリへ保存する。

- 現行の基準画像には安定した固定名を使う
- 差し替え前に旧版を意味の分かる名前で残す
- `final-v2`のような曖昧な連番より、`natural-light`、`brow-subtle`、`brow-bold`のように差分を示す
- 失敗版を現行ファイルへ残さない
- 生成元の既定保存ファイルを削除しない

完了時は、現行画像を絶対パスのMarkdown画像として表示し、保存先と主な変更点を簡潔に報告する。使用した生成モードと、重要なプロンプト方針も伝える。
