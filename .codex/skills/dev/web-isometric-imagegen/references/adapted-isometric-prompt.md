# Webアイソメ画像用・改変プロンプト

このファイルは、元のアイソメ・イエロー用スライドプロンプトの構造と制約を維持し、Web画像生成に必要な箇所だけを変更したものとする。画像生成時は全文を読み、該当案件の入力変数と具体的な場面を埋めて使う。短く要約した別プロンプトへ置き換えない。

主な変更は次の5点だけとする。

1. 出力単位をスライドからLP・Webサイト掲載画像へ変更する。
2. イエローを固定の青・緑パレットへ変更する。
3. 画像内文字を`text_mode`で切り替える。
4. 出力比率を`aspect_ratio`で指定する。
5. `$imagegen`による実生成、目視確認、修正生成を完了条件に加える。

## 入力変数

```text
【入力変数：テーマ】
依頼文、原稿、掲載セクションから導いた、この画像で伝える1つのテーマ。

【入力変数：text_mode】
none または include。未指定は none。

【入力変数：aspect_ratio】
明示指定、実際の掲載枠、4:3の順で決める。未指定かつ掲載枠不明は4:3。
```

ここから下は画像生成AIと制作エージェントへの指示であり、画像内には表示しない。この指示文、見出し、例文、ルール説明を画像内テキストとして使わない。

日本のニューエコノミー系ビジネスメディアを牽引するトップアートディレクター兼インフォグラフィックデザイナーとして、【入力変数：テーマ】に基づき、スマホ世代のビジネスパーソンの知的興奮を誘う、ビジュアル重視でハイセンスなLP・Webサイト用アイソメトリック画像を生成する。

## 出力時の絶対ルール

### 1. Markdown記号を出さない

- `text_mode: none`では、画像内に文字を一切描かない。
- `text_mode: include`でも、`#`、`*`、`**`などのMarkdown記号を含めない。
- 箇条書きが必要な場合も、行頭を`・`や`-`で始めず、通常の文章として書く。

### 2. プレーンテキストだけを使う

- `text_mode: include`で画像内に置く文字は、日本語と必要な短い英語のプレーンテキストだけにする。
- 確定していないコピー、数値、条件、ラベルを生成しない。

### 3. 1画像1メッセージ

- 各画像には、1つだけの主張、問い、結論を載せる。
- 情報量が多い場合は、迷わず画像を分割する。

### 4. ヘッダーまわり

- 上部へ帯状、タブ状のラベルを配置しない。
- `ISSUE 01`、`VECTOR`、`TRAP`などの内部ラベルを表示しない。
- `text_mode: include`でUI的な英語タグを入れる場合は、四隅や端へ小さなプレーンテキストとして置く。
- 枠線付きボックスや塗りつぶしたパネルを、見出し装飾として使わない。

### 5. 内部指示を漏らさない

- 画像内や端の小さな文字へ、`X`、`Y`、`LAYER`、`API`、`PARAM`、`DEBUG`などの内部語を出さない。
- 端に置ける英語は、意味が自然な短語だけに限定する。例: `INSIGHT`、`FOCUS`、`SIGNAL`、`NOISE`、`DECISION`、`PRIORITY`、`FRAME`、`WORKFLOW`。
- `text_mode: none`では、上記の限定語彙も含め、すべての文字を描かない。

## 世界観・ビジュアルOS

### トーン

- ミニマル、無彩色を土台にした、硬質で知的だが冷たすぎない表現にする。
- ニューエコノミー系の編集デザインとして、余白とコントラストで信頼を作る。
- 写真の情緒ではなく、比喩をアイコン化した理解の気持ちよさで気分を上げる。
- 青と緑は面積を支配する地色ではなく、意味を伝えるインジケーターとして使う。

### 比喩メタファーを必ず選ぶ

- 【入力変数：テーマ】から、最も直感的で自分ごと化しやすい比喩を1つだけ選ぶ。
- 候補は、都市、交通や路線・乗り換え、料理やレシピ・仕込み、登山やルート・装備、工場やライン・工程、図書館や索引・棚、ゲームやクエスト・UIとする。
- 以後の図解とイラストを、その比喩へ統一する。世界観を途中で変えない。
- 元の業務工程をそのまま描くようユーザーが明示した場合だけ、比喩ではなく実工程を統一モチーフとして使う。

### 視点・質感・イラスト方針

- 図解と場面表現を、ミニマルでやや丸みのあるアイソメトリック、等角投影で描く。
- 等角投影または正投影の3/4視点、約30度の軸を使い、強い遠近法や消失点を使わない。
- 黒から濃いグレーのクリーンで太い主線を使い、ラインアートと薄い影で構成する。
- 面を白またはライトグレー中心にし、情報構造が見える設計図感を作る。
- 影を主張させず、接地と前後関係を示す程度の柔らかい影にする。
- アイソメ図解には、視聴者の代理となる人間を1〜3人入れる。見る、迷う、選ぶ、確認する、進むなど行動が分かるポーズにする。
- ユーザーが人物なしを明示した場合だけ、人物を省略する。
- 写真は原則使わない。写真コラージュも使わない。
- 写真を使うよう明示された場合でも全面へ敷かず、小さな切り抜き素材1枚までに抑える。ただし、このスキルでは基本的に写真を使わない。

### グリッド

- 背景へ薄いグリッドを敷き、編集デザインとしての設計感を出す。
- グリッドはごく薄くし、主役や図解を邪魔しない。
- ヒーロー画像でもグリッドベースの構図を使うが、見える線を濃くしない。

## カラーパレット

デフォルトは次の青・緑・白パレットとし、カラーコードを生成プロンプトへすべて明記する。

- 背景: 白`#FFFFFF`または淡いクールブルーグレー`#F2F9FD`
- ベース要素: ライトグレーからホワイト
- 主線・主要文字: インク`#10161D`
- 補助線・補助文字: グレー`#4C565E`
- メインブルー: `#009FE8`
- ライトブルー: `#7DCCF3`
- メイングリーン: `#02A550`
- ライトグリーン: `#34B772`

色の意味を次へ固定する。

- メインブルー`#009FE8`: AI、整理、工程、システム、情報の流れ
- ライトブルー`#7DCCF3`: 接続線、補助工程、背景側の情報、弱い強調
- メイングリーン`#02A550`: 一次情報、人間の確認、承認、資産化、前進
- ライトグリーン`#34B772`: 確認後の状態、蓄積、成果側の補助面
- グレー: 未整理、背景、補助要素

青系と緑系以外の有彩色を使わない。シアン、レッド、イエロー、オレンジ、パープルなどを追加しない。発光や明度差が必要な場合も、上記4色と白の範囲だけで作る。

色を重要、焦点、選択、リスク、工程、確認、承認などの意味を持つインジケーターとして使う。装飾として散らさない。1画像の強調は1〜2箇所までにする。

ユーザーが具体的な色を明示した場合は、その色へ変更する。ユーザーが対象サイトから色を抽出するよう明示した場合は、そのときだけCSS、デザイン資料、既存画像から抽出する。どちらの明示もなければ、対象サイトから自動抽出せず、上記パレットを使う。

## タイポグラフィ

このセクションは`text_mode: include`の場合に適用する。`text_mode: none`では文字を描かず、HTML文字を配置できる余白として構図へ反映する。

### 見出し

- 極太ゴシックの日本語を使う。ImpactやヒラギノW8級の強さを目安にする。
- 画像内で最も大きい要素を、必ず見出しテキストにする。
- 見出しサイズを本文の少なくとも15倍にする。
- 見出しの行高が画像縦の約3分の1以上を占めるほど大きくする。
- 必要に応じて、見出しの近くへ細い英語短語を小さく添える。英語は限定語彙だけを使う。

### 本文

- 可読性の高いゴシック体を使う。
- 1ブロックを30〜40文字、最大2行の目安にする。
- 3行を超えそうなら画像を分割する。
- 本文の文字サイズを見出しの20%以下にし、強いジャンプ率を作る。

### 強調

- 重要な数字やキーワードを、青または緑のマーカーライン、色面、下線のいずれかで強調する。
- 強調色は、その語の意味に応じてカラーパレットの役割へ合わせる。
- 1画像あたりの強調を1〜2箇所までにする。

## コピーのエッジ

このセクションは`text_mode: include`の場合に適用する。

- キーポイントの見出しは、日本語の強い言い切りと短い英語1〜4語を組み合わせてよい。
- 英語は説明文ではなく、タグまたは断言として短く置く。
- 日本語は、自分ごと化できる痛み、境界線、矛盾、罠の命名を優先する。
- 次の方向性を参考にするが、そのまま引用せず毎回新規生成する。
  - 情報だけ増えて、前に進めない。`Overload`
  - その努力、違うゲームだ。`Wrong Game`
  - 迷うほど、手が止まる。`Decision Trap`
  - 一手だけ決める。`One Move`
- 英語候補は、`Overload`、`One Move`、`Wrong Game`、`Signal First`、`Choose One`、`Stop Scrolling`などとする。

## Web画像の構成: 3タイプ

各画像を、必ず次のいずれか1種類へ分類して構成する。TYPE名は内部判断にだけ使い、画像内へ表示しない。

### TYPE A: インパクト、文字重視

用途は、最重要メッセージ、強い結論、問いかけ、転換点とする。

- `text_mode: include`の場合だけ使う。
- 背景と巨大なテキストを中心にする。
- テキストを1〜2行に絞り、本文はなくてもよい。
- 背景を白または淡いクールブルーグレーにし、青または緑を少量だけ使う。
- アイソメ画像生成が依頼の中心であるため、Web画像として生成するときは小さくない主役アイソメモチーフを必ず併置する。文字だけにはしない。
- `text_mode: none`ではTYPE Aを選ばず、TYPE BまたはTYPE Cを使う。

### TYPE B: ビジュアル、イラスト重視

用途は、利用シーン、感情、ペイン、罠、危機、選択の描写とする。

- 写真ではなく、統一した比喩に沿うアイソメトリックのラインアート場面を使う。
- 視聴者代理の人物を1〜3人必ず入れる。人物なしが明示された場合だけ省略する。
- イラストをサムネイルサイズにせず、画像幅いっぱいに大きく配置する。
- 上下左右のいずれかを画面端まで到達させ、必要ならはみ出すほどトリミングする。
- `text_mode: include`では、見出しをイラストの上または周囲へ大きく置き、本文は短い一文だけ添える。
- `text_mode: none`では、HTML文字を置ける余白を残しながら、イラストだけでメッセージが伝わる構図にする。
- 罠や危機では構図を重くし、問題箇所や詰まりだけを青または緑で強調する。色の意味はパレット規則へ従う。

### TYPE C: ロジック、図解・イラスト重視

用途は、構造、比較、プロセス、仕組みとする。

- 長文説明ではなく、アイソメトリック図解またはミニマルな概念図で見せる。
- 青と緑以外の有彩色を使わない。
- 床面へ薄いグリッドを敷き、編集デザインの誌面感を維持する。
- `text_mode: include`では、四隅に意味が自然な短い英語を置いてよい。内部語は禁止する。
- `text_mode: none`では、四隅の英語も含め、文字を一切置かない。
- 視聴者代理の人物を1〜3人入れる。人物なしが明示された場合だけ省略する。

TYPE Cでは、必要に応じて次の代表レイアウトを使う。

- オーバービュー: 比喩世界を俯瞰し、重要エリアだけを青または緑で強調する。
- 分解図: 3層または3ステップに分け、関係だけを見せる。要素を増やしすぎない。
- ボトルネック図: 詰まり箇所だけを強調し、人物が困っている様子を入れる。
- フロー: 流れを線で示し、迷う分岐を明確に描く。
- A/B比較: 左右に並べ、違いを3点までにする。

## ヒーロー・OGPの特別指定

ヒーローまたはOGPは、シリーズ中で最もインパクトが強い画像にする。文字だけの画像は禁止し、TYPE BとTYPE Cのハイブリッド構成にする。

### デザイン・レイアウト

- スイス・スタイルとバウハウスを意識したグリッドベースにする。
- 統一した比喩に沿う主役のアイソメモチーフを必ず1つ置く。
- 視聴者代理の人物を1〜3人入れる。人物なしが明示された場合だけ省略する。
- タイトルを中央揃えにせず、四隅または上下いずれかの端へ寄せた非対称レイアウトにする。
- 上部へ帯状、タブ状のラベルを置かない。
- `text_mode: include`では、必要な場合だけ端へ限定語彙の英語短語を添える。
- `text_mode: none`では、HTMLのH1とサブコピーを置ける余白を確保する。
- PCとモバイルのトリミングで、主役、人物、流れが欠けない安全領域を確保する。

### タイトルコピー

このセクションは`text_mode: include`の場合に適用する。

- メインコピーを画像内で最大の文字にする。
- 抽象名詞1語だけのタイトルを禁止する。
- 読者のペインや境界線を1文で言い切る。
- 本文の少なくとも15倍の大きさにする。
- サブコピーは、読み進めると何が変わるかを1文で約束する。
- 必要に応じて、サブコピーの近くへ限定語彙の英語1〜4語を添える。

## 画像生成と確認

- ユーザーが画像生成を依頼した場合は、完成したプロンプトを`$imagegen`へ渡して実際に生成する。
- 配色変更の明示がなければ、プロンプトに`#009FE8`、`#7DCCF3`、`#02A550`、`#34B772`がすべて含まれることを確認する。
- 生成画像を開き、アイソメ投影、人物、主線、影、グリッド、色、文字、トリミングを目視確認する。
- 不備があれば、問題箇所を具体的に指定して修正生成する。
- 複数画像の依頼では、同じ比喩、主線、人物表現、配色の意味を維持し、指定順またはLP順に全件生成する。

## 最終確認

- すべての画像が、上記の世界観、カラールール、タイポグラフィ、TYPE A/B/Cのいずれかに従っているか。
- シリーズ全体が、ニューエコノミー系ビジネスメディアの特集記事をそのままWeb画像化したような、一貫したトーンになっているか。
- 1画像1メッセージになっているか。
- 比喩が1つに統一されているか。
- TYPE BとTYPE Cに、人物なしの明示がない限り、行動が分かる人物1〜3人が入っているか。
- TYPE Cで、比喩により【入力変数：テーマ】の位置づけと価値が直感的に伝わるか。
- 青と緑が意味を持つインジケーターとして使われ、装飾になっていないか。
- `text_mode: include`で、見出しが最大要素となり、文字量とジャンプ率が守られているか。
- `text_mode: none`で、意図しない文字が一切ないか。
- 内部指示、内部語、疑似UI、透かし状要素が混入していないか。
