## 実行ルール

- 作業開始時に適用される`AGENTS.md`、このplanの全文、`git status --short`を確認する。
- 最初の未完了Phaseを選び、checkboxへ着手する前に、そのPhaseの開始条件、完了状態、実行方針を読む。開始条件または実行方針が未確定なら作業を始めない。
- 選んだPhaseにある、依存解決済みの最初の`[ ]`項目から実行する。
- 実際のdiffと記載された検証結果を確認できた項目だけを`[x]`へ更新する。一部だけ完了した項目は完了部分と残作業へ分割する。
- Phase内の通常項目とGate条件が完了するまで次Phaseへ進まない。ユーザー承認や外部state変更を含むGateは、明示確認を得るまで`[ ]`のままにする。
- Cursor再判定Gateでは実fileとcontractから候補を再評価し、判断結果を対象Phaseの実行方針へ記録する。使用する場合だけ`cursor-agent-sprint-cli`で現在Sprintを`.codex/tmp/`へ詳細化する。
- Cursor workerにはplan更新、Gate通過、完了判断、commit、deployを任せない。main Codexがdiffを検収し、必要な検証を再実行する。
- 失敗、未検証、scope外変更、blockerは未完了のまま、該当Phaseの実行方針または「判断・結果ログ」へ事実と次の開始条件を記録する。
