# Jobs-To-Be-Done（JTBD）フレームワーク

## 概要

ユーザーが「雇う（hire）」仕事に焦点を当てる。製品の機能ではなく、ユーザーが達成したい進歩（progress）を理解する。

## JTBD文の構造

```
[状況] のとき、[動機] したい。そうすれば [期待する結果] になる。
```

例:
```
朝の通勤中に、短時間で業界ニュースを把握したい。そうすれば会議で的確な発言ができる。
```

## 分析の3層

### 1. Functional Job（機能的ジョブ）
ユーザーが達成したい具体的なタスク。
- 何を完了させたいか？
- どんな成果物が必要か？

### 2. Emotional Job（感情的ジョブ）
ユーザーが感じたい（避けたい）感情。
- 安心したい、不安を減らしたい
- 自信を持ちたい、恥をかきたくない

### 3. Social Job（社会的ジョブ）
他者からどう見られたいか。
- プロフェッショナルに見られたい
- チームに貢献していると認められたい

## ペイン/ゲイン分析

### ペイン（Pain）
現在の解決策で感じている不満・障害:
- 時間がかかりすぎる
- 複雑で分かりにくい
- コストが高い
- 信頼性が低い

### ゲイン（Gain）
理想的な解決策で得られる利益:
- 時間短縮
- 簡単・直感的
- コスト削減
- 品質向上
