# Codex Review 指示テンプレート

Codex モードの VERIFY Step 2a で `codex review` に渡すカスタムレビュー指示。
Claude オーケストレーターがこのテンプレートの `{変数}` を埋めて使用する。

## 実行コマンド

```bash
codex review --uncommitted "{埋め込み済みレビュー指示}" 2>/dev/null
```

## テンプレート

```
以下の仕様に基づいて変更をレビューしてください。

仕様概要: {仕様書の概要セクション — 1-2文に要約}
対象 Todo: {Todo の IMPL 内容 — 要約}

レビュー観点（重点的に確認）:
- diff 内のロジックバグ（因果関��が diff から説明できるもの）
- API・ライブラリの明確な誤用
- 型の不整合・スコープ内の矛盾
- 境界値・エッジケースの漏れ
- テストの網羅性（テストが書かれ��いる場合）

以下の観点は別のレ���ューワーが担当するため、指摘不要:
- セキュリティ・権限管理
- 仕様・ビジネス要件への適合性
- プロジェクト慣例・命名規則
- 既存機能への回帰リスク
```

## テンプレート変数の埋め方

| 変数 | 取得元 | 注意事項 |
|------|--------|----------|
| `{仕様書の概要セクション}` | 仕様書の「概要」 | 1-2文に要約する |
| `{Todo の IMPL 内容}` | 仕様書の対象 Todo | 要約する |

## 設計根拠

codex review は diff 局所で完結するロジックバグの検出に強く、以下は苦手:
- 仕様・ビジネス要件への適合性判断
- セキュリティ・権限管理
- テスト不足の検出
- 既存機能へ���暗黙的な影響
