## タスクリストの実行方式タグ

各タスクにオプションで実行方式を付与できる:

```
`[実行方式:モデル:サブエージェント]`
```

### 記法ルール

| 記法                 | 意味                                           |
| -------------------- | ---------------------------------------------- |
| （タグなし）         | メインスレッドで実行（デフォルト）             |
| `[BG:haiku:Explore]` | バックグラウンドで haiku、Explore エージェント |
| `[PARALLEL]`         | 次の複数タスクを並列実行                       |

### モデル選択基準

| モデル | 用途                                   |
| ------ | -------------------------------------- |
| haiku  | 単純な検索・ファイル読み込み・定型作業 |
| sonnet | 中程度の複雑さの実装・分析             |
| opus   | 複雑な設計・重要な判断                 |

### サブエージェントタイプ

| タイプ          | 用途             |
| --------------- | ---------------- |
| Explore         | コードベース探索 |
| general-purpose | 汎用タスク       |
| code-simplifier | コード整理       |

## ワークフロー分類

各タスクの `ワークフロー` フィールドには以下のいずれかを指定する:

| ワークフロー | 対象                                               | 判定基準                                       |
| ------------ | -------------------------------------------------- | ---------------------------------------------- |
| **TDD**      | ビジネスロジック、バリデーション、計算、データ変換 | 入出力が明確に定義でき、アサーションで検証可能 |
| **E2E**      | UI コンポーネント、レイアウト、ユーザー操作        | 視覚的確認または UX/UI 判断が必要              |
| **TASK**     | 設定ファイル、環境構築、ドキュメント               | テスト不要、コマンド実行結果で検証可能         |

判定フロー:

```
入出力が明確に定義できる？
    ├─ YES → TDD
    └─ NO → 視覚的確認が必要？
              ├─ YES → E2E
              └─ NO → TASK
```
