# 仕様書テンプレート

仕様書 `docs/PLAN/{YYMMDD}_{slug}.md` は以下の構成で作成する。

---

## 必須セクション

```markdown
# {タイトル}

## 注意書き
この仕様書に基づいて実行を進める際は、タスクリストのチェックを
常に更新すること。

**実装エージェントへの注記**: この仕様書は自己完結している。
「参照すべきファイル」に記載されたファイルを読んでから着手すること。

## 概要
[1-2 文で何を達成するか]

## 背景
[なぜこの変更が必要か、設計判断の理由]

## 設計決定事項
[確定した意思決定を表形式で列挙]

| # | トピック | 決定 | 根拠 |
|---|----------|------|------|
| 1 | ... | ... | ... |

## アーキテクチャ詳細
[データフロー図、構造定義、仕様値など実装に必要な詳細]
[ASCII 図、JSON スキーマ、型定義などを含める]

## 変更対象ファイルと影響範囲

### 変更するファイル
| ファイル | 変更内容 | 影響 |
|---------|----------|------|

### 新規作成ファイル
| ファイル | 内容 |
|---------|------|

### 変更しないファイル
| ファイル | 理由 |
|---------|------|

## 参照すべきファイル

実装着手前に必ず読むこと。

### コードベース内
| ファイル | 目的 |
|---------|------|

### 参照資料（docs/PLAN/{YYMMDD}_{slug}/references/ にコピー済み）
| ファイル | 目的 |
|---------|------|
[外部参照がない場合はこのサブセクション自体を省略]

## タスクリスト

### 依存関係図
[ASCII で依存関係を図示]
```
Todo 1 → Todo 2 → Todo 3
Todo 4 → (Todo 5 | Todo 6)  ← 並行実行可
```

### Phase N: {フェーズ名}
> {フェーズの説明、並列可否などの注記}

- [ ] **Todo N: {タイトル}**
  - **対象**: {ファイルパス}
  - **内容**: {何を — 具体的なアクション}
  - **実装詳細**: {どうやって — 既存パターン参照、コード例}
  - **検証**: {テストコマンドまたは確認手順}
  - **ワークフロー**: TDD / E2E / TASK
  - **依存**: {先行タスク — なければ「なし」}

## 残存リスク
| リスク | 影響 | 緩和策 |
|--------|------|--------|
```

---

## タスクリストの実行方式タグ

各タスクにオプションで実行方式を付与できる:

```
`[実行方式:モデル:サブエージェント]`
```

### 記法ルール

| 記法 | 意味 |
|------|------|
| （タグなし） | メインスレッドで実行（デフォルト） |
| `[BG:haiku:Explore]` | バックグラウンドで haiku、Explore エージェント |
| `[PARALLEL]` | 次の複数タスクを並列実行 |

### モデル選択基準

| モデル | 用途 |
|--------|------|
| haiku | 単純な検索・ファイル読み込み・定型作業 |
| sonnet | 中程度の複雑さの実装・分析 |
| opus | 複雑な設計・重要な判断 |

### サブエージェントタイプ

| タイプ | 用途 |
|--------|------|
| Explore | コードベース探索 |
| general-purpose | 汎用タスク |
| code-simplifier | コード整理 |

---

## ワークフロー分類

各タスクの `ワークフロー` フィールドには以下のいずれかを指定する:

| ワークフロー | 対象 | 判定基準 |
|-------------|------|----------|
| **TDD** | ビジネスロジック、バリデーション、計算、データ変換 | 入出力が明確に定義でき、アサーションで検証可能 |
| **E2E** | UI コンポーネント、レイアウト、ユーザー操作 | 視覚的確認または UX/UI 判断が必要 |
| **TASK** | 設定ファイル、環境構築、ドキュメント | テスト不要、コマンド実行結果で検証可能 |

判定フロー:
```
入出力が明確に定義できる？
    ├─ YES → TDD
    └─ NO → 視覚的確認が必要？
              ├─ YES → E2E
              └─ NO → TASK
```
