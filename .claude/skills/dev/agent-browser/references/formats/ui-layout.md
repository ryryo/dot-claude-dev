## レポートフォーマット: ui-layout（表示・レイアウト確認）

### 各ステップで必ず実行するコマンド
- 各状態で `snapshot`（フルツリー）を実行し、**出力をそのまま引用**
- 主要な状態で `screenshot --full {SCREENSHOT_DIR}/{NN}-{description}.png` を保存
- 視覚的な問題点があれば該当要素の `get text @ref` で内容を取得

### レポート構造

```
## 1. ページ構造
[snapshot のフルツリー出力を引用]

## 2. スクリーンショット
- {パス}: {説明}

## 3. 問題点
- [具体的な問題と該当要素の出力引用]
```
