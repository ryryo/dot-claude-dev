# Linter スクリプトテンプレート指示

> **読み込み条件**: type === "linter" の場合
> **対応ランタイム**: Node.js, Bash

---

## 目的

コード品質チェック・リント処理を行うスクリプトを生成する。

---

## AIへの実装指示

### 必須機能

1. **リント実行**
   - 指定ファイル/ディレクトリのチェック
   - 設定ファイルの読み込み
   - ルールの適用

2. **自動修正**
   - 修正可能な問題の自動修正
   - 修正内容のプレビュー

3. **レポート出力**
   - エラー/警告の一覧
   - ファイル別サマリー
   - 複数フォーマット対応（JSON/text/stylish）

4. **CI統合**
   - 終了コードによる結果通知
   - 差分のみチェック

### 推奨依存関係

| ランタイム | 推奨パッケージ |
|------------|----------------|
| Node.js | eslint, biome |
| Python | ruff, flake8, pylint |
| Bash | shellcheck |

### 環境変数

- `{{LINT_CONFIG}}`: 設定ファイルパス
- `{{LINT_IGNORE}}`: 無視パターン

### 引数仕様

| 引数 | 必須 | 説明 |
|------|------|------|
| --path | ○ | チェック対象パス |
| --fix | × | 自動修正を実行 |
| --format | × | 出力フォーマット |
| --config | × | 設定ファイル |
| --quiet | × | エラーのみ出力 |

### 出力形式

- リント結果を指定フォーマットで出力

---

## コード構造の指示

AIは以下の構造でコードを生成してください:

```
1. インポート
   - リンターライブラリ
   - ファイル操作

2. 設定
   - デフォルトルール
   - 終了コード定数

3. ユーティリティ関数
   - loadConfig(): 設定読み込み
   - runLinter(): リント実行
   - autoFix(): 自動修正
   - formatResults(): 結果整形

4. メイン処理
   - 引数パース
   - 設定読み込み
   - リント実行
   - 結果出力
```

---

## 品質基準

AIは以下の基準を満たすコードを生成してください:

- [ ] 設定ファイルの自動検出
- [ ] 部分的なチェック対応
- [ ] キャッシュによる高速化
- [ ] エラー位置の正確な報告
- [ ] カスタムルール対応
