# Universal スクリプトテンプレート指示

> **読み込み条件**: type === "universal" の場合
> **対応ランタイム**: Node.js, Python, Bash, TypeScript

---

## 目的

特定のカテゴリに分類されない、任意の目的を持つスクリプトを生成する。

---

## AIへの実装指示

### 設計原則

1. **目的駆動**
   - ユーザー要求から目的を明確に抽出
   - 最小限の機能で目的を達成

2. **柔軟性**
   - 拡張しやすい構造
   - 設定可能なパラメータ

3. **再利用性**
   - 単一責務の関数
   - 明確なインターフェース

### 実装フロー

AIは以下のフローで実装を進めてください:

```
1. 要件分析
   - 何を達成したいか
   - 入力は何か
   - 出力は何か
   - 外部依存は何か

2. ランタイム選択
   - 要件に最適なランタイムを選択
   - 必要な依存関係を特定

3. 設計
   - 関数分割
   - データフロー設計
   - エラーハンドリング設計

4. 実装
   - 基本テンプレートを基に
   - 目的に特化した処理を追加
```

### 引数仕様

AIは要件に基づいて適切な引数を設計してください:

| 引数タイプ | 用途 |
|------------|------|
| --input / -i | 入力（必要な場合） |
| --output / -o | 出力先（必要な場合） |
| --config / -c | 設定ファイル（必要な場合） |
| --verbose / -v | 詳細出力 |
| --dry-run | 実行せずに内容表示 |
| --help / -h | ヘルプ表示 |

## コード構造の指示

AIは要件に応じて最適な構造を選択してください:

### 単純なスクリプトの場合

```
1. インポート
2. 定数
3. メイン処理（直接実行）
```

### 中程度の複雑さの場合

```
1. インポート
2. 定数
3. ユーティリティ関数
4. メイン処理
5. エントリポイント
```

### 複雑なスクリプトの場合

```
1. インポート
2. 型定義（TypeScriptの場合）
3. 定数
4. 設定クラス/オブジェクト
5. ユーティリティ関数
6. コア処理関数
7. メイン処理
8. エントリポイント
```

---

## 品質基準

AIは以下の基準を満たすコードを生成してください:

- [ ] 目的に対して過不足のない実装
- [ ] 明確なエラーメッセージ
- [ ] 適切な終了コード
- [ ] 必要十分なログ出力
- [ ] 拡張可能な構造
- [ ] ドキュメント（ヘルプ表示）の充実

---

## 注意事項

- 具体的な例ではなく、要件に基づいて実装を考える
- 不明点があれば確認を求める
- 過剰な機能追加を避ける
- 将来の拡張性より現在の要件を優先
